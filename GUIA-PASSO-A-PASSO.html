Como publicar o formulário Mentor QA na sua conta

Este guia reproduz, passo a passo, tudo que foi feito para colocar o formulário no ar: conectar a uma planilha Google (para receber as respostas) e hospedar a página no GitHub Pages. Não precisa saber programar — é copiar, colar e clicar.

Tempo: ~20 minutos Custo: gratuito Precisa: conta Google + conta GitHub
🎯
O que você vai ter no final: um link (ex.: https://seu-usuario.github.io/agente-mentor-qa/) que qualquer pessoa abre, responde o check-in de QA, e as respostas caem organizadas numa planilha sua. Depois, você roda o agente Mentor QA sobre cada resposta para gerar o PDI individual.

🧭 Como as peças se encaixam

1
A página (GitHub Pages)

Um arquivo HTML hospedado de graça no GitHub. É o formulário que a pessoa preenche.

2
O carteiro (Google Apps Script)

Um mini-script gratuito que recebe as respostas e escreve na planilha. Fica "no ar" numa URL própria.

3
A caixa (Planilha Google)

Onde cada resposta vira uma linha. É de onde você lê os dados depois.

As 4 etapas

  1. Pegar os arquivos do projeto2 min
  2. Criar a planilha e o script (Google)8 min
  3. Colar a URL no formulário2 min
  4. Publicar no GitHub Pages6 min
1

Pegar os arquivos do projeto

Você precisa de 4 arquivos: o formulário, o script, e dois auxiliares.

  1. Reúna estes 4 arquivos numa pasta

    • index.html — o formulário (a página em si).
    • apps-script.gs — o script que grava na planilha.
    • README.md — instruções (opcional, mas bom manter).
    • .nojekyll — um arquivo vazio que o GitHub Pages precisa. Não apague.
    💡
    Quem te enviou este guia provavelmente enviou junto a pasta agente-mentor-qa com esses arquivos. Se não, peça a ela os 4 arquivos.
2

Criar a planilha e o script no Google

Aqui você monta a "caixa" que recebe as respostas. É a parte mais longa — vá com calma.

  1. Crie uma planilha nova

    Na barra de endereços do navegador, digite:

    sheets.new

    Abre uma planilha em branco. Dê um nome, ex.: "Mentor QA — Respostas".

  2. Abra o editor de script

    No menu da planilha: Extensões → Apps Script. Abre uma aba nova com um editor de código.

  3. Cole o código do script

    No editor, apague o que estiver lá (normalmente um function myFunction() {} vazio). Abra o arquivo apps-script.gs num bloco de notas, selecione tudo (Ctrl+A), copie (Ctrl+C) e cole no editor (Ctrl+V).

    Clique no ícone de disquete (Salvar projeto) ou Ctrl+S.

    💡
    Copie do arquivo, não de uma imagem — assim não erra nada. Você não precisa editar o código; ele já sabe usar a planilha à qual está ligado.

  4. Rode a função que prepara a planilha

    Na barra cinza no topo do editor, há um menu de funções. Selecione prepararPlanilha e clique em ▶ Executar.

    Isso cria a aba "Respostas" com o cabeçalho já formatado (em verde), automaticamente.

  5. Autorize as permissões (a tela que assusta, mas é normal)

    Na primeira execução, o Google pede autorização. Siga exatamente:

    • Clique em Revisar permissões → escolha sua conta Google.
    • Vai aparecer uma tela amarela como esta:
    accounts.google.com
    ⚠️ O Google não verificou este app

    O app está solicitando acesso a informações confidenciais na sua Conta do Google. Não é recomendado usá-lo até que o desenvolvedor faça a verificação com o Google.

    Avançado

    • Clique em Avançado (canto inferior).
    • Clique no link Acessar (nome do projeto) (não seguro).
    • Na tela de permissões, clique em Permitir.
    🛡️
    Por que esse aviso aparece e por que é seguro: o Google mostra isso para todo script pessoal. O "desenvolvedor" citado é você mesmo — você está autorizando o seu próprio código a mexer na sua própria planilha. Ele não acessa mais nada da sua conta.

  6. Confira que deu certo

    No rodapé do editor deve aparecer Execução concluída (sem erro vermelho). Volte à planilha: apareceu a aba "Respostas" com um cabeçalho verde (Nome, E-mail, Senioridade…)? Então esta parte está pronta. ✅

  7. Publique o script como "App da Web"

    É isso que deixa o script "no ar" para o formulário conseguir enviar. No editor, canto superior direito:

    • Implantar → Nova implantação.
    • Clique na engrenagem ⚙ ao lado de "Selecionar tipo" → escolha App da Web.
    • Preencha:
      • Executar como: Eu (seu e-mail).
      • Quem pode acessar: Qualquer pessoa.
    • Clique em Implantar (autorize de novo se pedir).
    ⚠️
    O passo mais fácil de errar: "Quem pode acessar" tem que ser "Qualquer pessoa" — e não "Qualquer pessoa com Conta do Google". Se ficar restrito, o formulário não consegue enviar. O script só grava linhas; ninguém vê sua planilha por isso.

  8. Copie a URL do app

    No fim, aparece uma URL do app da Web, longa e terminada em /exec:

    https://script.google.com/macros/s/AKfycb…longo…/exec

    Clique em Copiar. Guarde essa URL — ela é usada na Etapa 3.

    🔒
    Essa URL não é uma senha. Ela só serve para enviar dados ao seu script (que apenas adiciona linhas). Pode guardá-la sem medo. O que nunca se compartilha é a senha da sua conta Google.
3

Colar a URL no formulário

Agora você conecta o formulário à sua "caixa" do Google.

  1. Abra o index.html num editor de texto

    Bloco de Notas ou VS Code servem. Procure (com Ctrl+F) por APPS_SCRIPT_URL. Você vai achar esta linha:

    var APPS_SCRIPT_URL = "COLE_AQUI_A_URL_DO_APPS_SCRIPT";

  2. Substitua pelo seu link

    Troque o COLE_AQUI_A_URL_DO_APPS_SCRIPT pela URL que você copiou (a que termina em /exec). Mantenha as aspas. Deve ficar assim:

    var APPS_SCRIPT_URL = "https://script.google.com/macros/s/AKfycb…/exec";

    Salve o arquivo (Ctrl+S).

    💡
    Enquanto a URL não estiver colada, o formulário ainda funciona em "modo offline": mostra o resumo das respostas para copiar, mas não envia ao Google. Depois de colar, ele passa a enviar de verdade.

  3. Teste na sua máquina (recomendado)

    Dê um duplo-clique no index.html para abri-lo no navegador. Preencha um teste (os campos obrigatórios: nome, e-mail, senioridade, objetivo, as 5 situações e o consentimento) e clique em Enviar e receber meu PDI.

    Deve aparecer "Recebido! Já está com a gente." Depois, abra sua planilha (aba "Respostas") e confira: apareceu uma linha nova com os dados? Se sim, a conexão está funcionando. 🎉

4

Publicar no GitHub Pages

O último passo: colocar a página no ar num link público, de graça.

  1. Crie um repositório no GitHub

    Em github.com/new, crie um repositório novo. Sugestão de nome: agente-mentor-qa. Deixe Público (o GitHub Pages gratuito exige repositório público).

    👀
    Ciente do que fica público: a URL do seu Apps Script ficará visível no código do repositório. Ela não dá acesso à sua planilha nem à sua conta — mas alguém que a descubra poderia enviar linhas de spam para a planilha. Para uso interno, o risco é baixo. Se incomodar, dá para adicionar um código de acesso ao formulário depois.

  2. Suba os 4 arquivos

    Opção simples (sem linha de comando): na página do repositório recém-criado, clique em uploading an existing file e arraste os 4 arquivos (index.html, apps-script.gs, README.md, .nojekyll). Clique em Commit changes.

    Opção linha de comando (se você usa git), dentro da pasta do projeto:

    git init git add . git commit -m "Formulário Mentor QA" git branch -M main git remote add origin https://github.com/SEU-USUARIO/agente-mentor-qa.git git push -u origin main
    📄
    Confira que o .nojekyll foi enviado. Por começar com ponto, alguns sistemas o escondem. Sem ele, o GitHub Pages pode não publicar a página corretamente.

  3. Ligue o GitHub Pages

    • No repositório: Settings → menu lateral Pages.
    • Em Source, escolha Deploy from a branch.
    • Em Branch, selecione main e a pasta / (root) → Save.

    Aguarde cerca de 1 minuto. O link aparece no topo dessa mesma página:

    https://SEU-USUARIO.github.io/agente-mentor-qa/

  4. Teste final no link real

    Abra o link do Pages, envie uma resposta de teste e confirme que a linha caiu na planilha. Este é o teste no ambiente de verdade (em https://). Pronto — está no ar! 🚀

🔑

Código de acesso (só quem você convida responde)

O formulário é por convite: quem não digitar o código certo não consegue enviar.

  1. Escolha o seu código

    Já vem MENTORQA2026 de fábrica. Você pode manter ou trocar por outro. Precisa ser igual nos dois arquivos.

  2. No index.html

    Procure (Ctrl+F) por ACCESS_CODE e ajuste o valor.

    var ACCESS_CODE = "MENTORQA2026";

  3. No apps-script.gs

    Procure por CODIGO_ACESSO e coloque o mesmo valor.

    var CODIGO_ACESSO = 'MENTORQA2026';
    ⚠️
    Os dois têm que ser idênticos. Se trocar depois de já ter publicado o script, republique (veja "Se algo não funcionar" → "Editei o script depois de publicar").

  4. Passe o código só para quem você quer

    Envie no convite (e-mail, chat). Quem não tiver o código verá "Código de acesso incorreto" e não consegue enviar.

Se algo não funcionar

  1. Enviei, mas não aparece linha na planilha

    Quase sempre é o "Quem pode acessar" da publicação: tem que ser "Qualquer pessoa". Volte no editor do script → Implantar → Gerenciar implantações → edite (lápis) → corrija o acesso → Implantar.

  2. Editei o script depois de publicar

    Toda vez que mudar o apps-script.gs, republique: Implantar → Gerenciar implantações → editar (lápis) → Versão: Nova versão → Implantar. Assim a mesma URL passa a valer o novo código (a URL não muda). Se criar uma implantação nova do zero, a URL muda e você precisa atualizar o index.html.

  3. A página do GitHub Pages não abre / aparece em branco

    Confirme que o .nojekyll está no repositório e que o Pages está apontando para main / (root). Depois de ligar, o primeiro deploy leva 1–2 minutos.

  4. A planilha começou a receber linhas de lixo (spam)

    Como a URL é pública, isso é possível. A solução é adicionar um código de acesso ao formulário (uma palavra secreta que só quem você convida recebe). Peça a quem montou o projeto para incluir essa proteção — é uma alteração pequena.
